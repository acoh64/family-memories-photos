# Cape Cod Cohen — Family Memories

A static family photo and memories site hosted on GitHub Pages.

## Quick Start

1. Push this repo to GitHub.
2. In your repo settings, enable **GitHub Pages** from the `main` branch (root `/`).
3. Your site will be live at `https://<username>.github.io/<repo-name>/`.

## Adding a New Memory

1. **Prepare your photo(s)** — add image files to the `photos/` folder.
2. **Open the Add Memory page** on the live site (or `add.html` locally).
3. Fill out the form with a title, date, description, photo filenames, and tags.
4. Click **Generate JSON** and copy the output.
5. Open `data/memories.json` and paste the new entry into the array (add a comma after the preceding entry).
6. Commit and push. The site updates automatically.

### Memory JSON Schema

Each entry in `data/memories.json` follows this structure:

```json
{
  "id": "unique-slug",
  "title": "Memory Title",
  "description": "The full story...",
  "date": "YYYY-MM-DD",
  "photos": ["filename.jpg"],
  "tags": ["tag1", "tag2"]
}
```

- **id** — URL-safe slug (auto-generated by the form)
- **photos** — Array of filenames that live in the `photos/` directory
- **tags** — Used for filtering in the gallery

```
├── add.html            Helper form for generating entries
├── css/style.css       All styles
├── js/app.js           Gallery, lightbox, filtering, search
├── js/add.js           Add-memory form logic
├── data/memories.json  Memory entries
├── photos/             Image files
└── README.md
```

## Local Development

No build step required. Open `index.html` in a browser, or use any static server:

```bash
npx serve .
```

> **Note:** Fetching `memories.json` via `file://` may be blocked by CORS.
> Use a local server for the best experience.
